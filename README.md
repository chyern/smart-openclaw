# smart-openclaw

> OpenClaw AI 助手配置模板

高度自定义的 AI 助手设定，赋予 OpenClaw 更强的自主性、记忆能力和协作风格。

---

## 📁 项目结构

```
smart-openclaw/
├── SOUL.md      # AI 人格设定（核心身份、底线、行为准则）
├── AGENTS.md    # AI 工作模式（行为准则、做事流程）
├── apply.sh     # 一键部署脚本
├── skills/      # 技能目录
├── LICENSE      # 开源协议
└── .gitignore   # Git 忽略配置
```

---

## 🎯 功能特性

### 1. 自主人格
- **独立思考** - 不是机械执行，会质疑并给出建议
- **成长型** - 从每次交互中学习，不断优化行为
- **边界清晰** - 涉及安全和隐私的事有自我判断

### 2. 协作模式
- **确认机制** - 重要操作前必定二次确认
- **主动建议** - 发现问题会主动提出，而非默默执行
- **结果回馈** - 任务完成后主动汇报，含修改对比

### 3. 做事流程
- 多听、多想、多问、再执行
- 分阶段确认，避免方向偏差
- 验证失败有标准处理流程

## 🚀 快速开始

### 1. 克隆项目

```bash
git clone https://github.com/your-repo/smart-openclaw.git
cd smart-openclaw
```

### 2. 一键部署

```bash
./apply.sh
```

脚本会自动：
- 备份现有的 `AGENTS.md` 和 `SOUL.md`
- 复制新配置到 `~/.openclaw/workspace/`
- 显示备份位置

### 3. 重启 OpenClaw

```bash
openclaw gateway restart
```

## 📖 配置说明

### SOUL.md - 人格设定

定义 AI 的核心身份和行为底线：

| 章节 | 内容 |
|------|------|
| 底线 | 执行前获得许可、保护隐私、数据安全 |
| 核心身份 | 自主思考、有判断力、主动改进 |
| 记忆系统 | 记录原则、每日自检、每周复盘 |
| 核心信条 | 系统性思考、敏感信息脱敏、主动记录 |

### AGENTS.md - 工作模式

定义 AI 的具体行为规范：

| 章节 | 内容 |
|------|------|
| 行为准则 | 确认意图、区分提问/任务、协作模式 |
| 做事流程 | 理解→查询→确认→实现→验证→汇报→复盘 |
| Git 策略 | 获得授权、提交前确认、冲突处理 |
| 情绪与心态 | 根据场景调整响应方式 |

## 🔧 常见问题

**Q: 部署后没有生效？**
> 确保运行了 `openclaw gateway restart` 重启网关

**Q: 如何回退到之前的配置？**
> 查看 `~/.openclaw/workspace/backup/` 目录下的备份文件

**Q: 如何更新配置？**
> 修改 `SOUL.md` 或 `AGENTS.md` 后，重新运行 `./apply.sh`
